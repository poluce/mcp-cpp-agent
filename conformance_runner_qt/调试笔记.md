## 已解决的关键技术问题

### 1. HTTP/SSE 握手 Race Condition 竞态时序 Bug

- **问题**：在原版设计中，客户端调用 `connectHttpAndWait` 开启网络后，底层物理 SSE Stream 还没来得及接收到服务端的第一个 `endpoint` 报文（以获取会话特定的 `sessionId`），就瞬间发出了 `initialize` 消息。此时由于 POST URL 不含有 `sessionId`，被服务端直接返回 `400 Bad Request` 默默拒绝，导致后续交互全部超时。
- **涉及文件**：`mcp_qt_transport/src/QtHttpSseWorker.cpp`、`mcp_qt_transport/src/QtHttpSseWorker.h`
- **已修复**：为 `QtHttpSseWorker` 引入了**待发送消息缓冲队列**机制（`m_pendingMessages`）。在收到 `endpoint` 报文且成功解析出 `sessionId` 路由地址之前，所有 POST 请求会自动缓存；在 endpoint 就绪的第一时间，自动冲刷发送全部缓存消息。此举成功解决了握手超时问题，令 `sse-retry` 等场景完美通过。

### 2. MinGW 系统库链接缺失（bcrypt）

- **问题**：在去除 `libcurl` 依赖后，`McpOAuthClient`（用于 PKCE 运算）在 Windows 平台采用系统的 `bcrypt` 实现 SHA-256 加密。在 MSVC 下可通过 `#pragma comment(lib, "bcrypt.lib")` 自动链接，但在用户的 MinGW (g++) 编译链下该 pragma 无效，导致最终链接生成可执行程序时大量报错未定义的 `BCryptOpenAlgorithmProvider` 等引用。
- **涉及文件**：`src/core/CMakeLists.txt`
- **已修复**：在 CMake 脚本中添加了平台感知链接：若在 Windows 下编译，强制为 `mcp_core` 链接 `bcrypt` 静态库，解决了 MinGW 下的链接问题。

### 3. 测试场景工具名不匹配

- **问题**：在 `ScenarioHandlersAuthQt.cpp` 原代码中，一些场景（如 `runToolsCall` 等）写死了调用 `"add_numbers"` 或 `"test-tool"` 等工具，但这些工具跟 Mock 服务端 `test_mcp_server.js` 中导出的实际工具（如 `"calculate_add"`）不一致，导致工具调用失败。
- **已修复**：将测试场景中调用的工具统一对齐为 Mock 服务端导出的真实工具（`"calculate_add"` 和 `"get_system_time"`），打通了整个测试流程。

### 4. 彻底的纯 Qt 化改造与冗余清理

- **已删除**：删除了所有纯 C++ 场景下的单元测试（原 `tests/` 目录）和合规测试（原 `conformance_runner/` 目录），并移除了所有对 `libcurl` 库的下载和依赖，使项目彻底变为了零 curl 外部依赖的纯 Qt 项目。
- **保留并迁移**：将合规测试客户端专需的控制台通信管道 `ConsoleStdioTransport.h/cpp` 迁移至 `conformance_runner_qt` 本地以支持 `tools/list` 场景；同时在 `tests_qt/` 目录下放置了指向本地 `common.h` 的重定向头文件，在不修改原有测试代码的情况下平滑理顺了编译。

## 架构说明

```
conformance_runner_qt/
├── CMakeLists.txt              # Qt 版本合规测试构建配置（仅链接 mcp_qt_client/mcp_core（现均在 src/ 下）/Qt6）
├── main_qt.cpp                 # 入口：解析环境变量 → 创建 RunnerConfig → 调用 runScenario
├── ScenarioHandlersAuthQt.cpp  # 所有场景处理函数（使用 McpQtClient API 或底层 HttpSseTransport）
├── ConsoleStdioTransport.h/cpp # 本地控制台 Stdio 管道传输实现（供 runToolsList 使用）
├── RunnerConfig.h/cpp          # 命令行/环境变量配置解析
├── ScenarioContext.h/cpp       # 场景上下文模型
├── ScenarioRegistry.h/cpp      # 场景注册表
└── 调试笔记.md                  # 本文件
```
